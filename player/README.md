# 마비노기 MML 재생기 샘플 v2.4

공개용 단일 페이지 샘플입니다.
외부 CDN이나 온라인 모듈을 사용하지 않고, 필요한 코드는 `js/` 폴더의 로컬 파일로 분리되어 있습니다.

## 구성

```text
mabinogi_mml_public/
├─ index.html
├─ styles.css
├─ README.md
├─ favicon.ico
├─ assets/
│  ├─ favicon.ico
│  ├─ Roland_SC-55.sf2
│  └─ default-sf2-base64.js
└─ js/
   ├─ utils.js
   ├─ midi-to-mml.js
   ├─ mml-parser.js
   ├─ mml-optimizer.js
   ├─ sf2-sampler.js
   └─ app.js
```

## 기능

- 채널별 음색 설정: MML 6개 채널마다 SF2 프리셋을 따로 선택해 재생할 수 있습니다. 자동 음색과 사용자가 저장한 음색 프리셋을 바로 적용할 수 있습니다.

- MML 재생 / 정지
- 반복 재생 체크박스, 기본값 꺼짐
- 볼륨 / 배속 / 반복 / 테마 상태를 브라우저 로컬 저장소에 보관
  - 페이지를 다시 열면 마지막 설정을 최대한 복원합니다.
  - 저장값이 없으면 기본값은 볼륨 100%, 배속 1.00x, 반복 꺼짐, 밝은 테마입니다.
  - 테마 전환은 제목 오른쪽의 작은 색상 버튼으로 조작합니다.
- 시작 지점으로 되돌리기 버튼
- 재생 위치 확인 및 슬라이더 이동
  - 재생 조작 UI는 진행 슬라이더가 가장 넓게 보이도록 2단 배치로 구성했습니다.
- 재생 중 슬라이더 클릭/이동으로 구간 이동
- 재생 슬라이더에 템포 변화 지점 표시
  - 현재 적용 중인 템포는 초록색으로 크게 표시
- 볼륨 조절, 기본값 100%
- `mid`, `midi`, `txt`, `md` 파일 불러오기
  - `mid`, `midi`: 파일 분석 후 MIDI 변환 설정 다이얼로그를 표시합니다.
    - 왼쪽에서 MML 채널별 `자동/고음/저음/비트`, `겹침 병합 방식`을 설정합니다.
    - 오른쪽에서 현재 선택한 MML 채널이 가져올 악기를 고릅니다.
    - 악기 목록에는 전부 선택/전부 해제 버튼과 악기별 듣기 버튼이 있습니다.
    - 기본값은 표시 가능한 악기를 모두 선택한 상태입니다.
    - 한 MML 채널에 여러 악기를 선택할 수 있고, 여러 MML 채널이 같은 악기를 선택할 수도 있습니다.
    - 채널 목록에는 악기 이름 대신 선택된 악기 개수를 표시하고, 악기 목록에는 해당 악기가 선택된 MML 채널을 표시합니다.
    - 단, 원본 MIDI 노트 하나는 전체 MML 채널 중 한 곳에만 배치됩니다.
    - 일반 채널 역할은 일반 악기만, 비트 역할은 비트 악기만 선택할 수 있습니다.
    - 드럼, 북, 공, 스네어, 심벌즈 등 비트 악기는 비트 그룹으로 분류합니다.
    - 64분음표 양자화 기준으로 같은 원본 연주 그룹 안의 완전 중복 노트는 병합합니다. 같은 악기명이 여러 채널에 있어도 선택은 하나로 묶되, 중복 병합은 채널을 넘어 수행하지 않습니다.
    - Export 채널 수는 1~6개 중 선택하며 기본값은 3개입니다.
    - 기본 역할은 1번 채널 `고음`, 3번 채널 `저음`입니다. 기본 겹침 병합은 멜로디와 화음2가 `절반`, 나머지가 `모두`입니다.
    - 비트 악기 그룹이 없으면 역할 선택에서 `비트` 항목을 숨기고 안내 문구를 표시합니다.
  - `txt`: `MML@...;` 형식일 때 자동 최적화하여 전체 MML에 반영
  - 채널 내부 문법 오류로 최적화할 수 없는 TXT 파일은 형식만 맞으면 원본을 불러오고, 최적화 생략 안내를 표시합니다.
- 기본 SF2 내장 사용: `Roland_SC-55.sf2`
- 사용자 SF2 선택 및 기본 사운드 복귀
- 채널별 재생 음색 설정
  - 편집 편의 기능 줄 오른쪽의 `음색` 버튼에서 멜로디/화음1~5가 사용할 SF2 프리셋을 각각 선택합니다.
  - Dialog의 `음색 프리셋` 콤보박스 첫 항목은 `자동 음색`입니다. 최근 MIDI 변환 음색이 있으면 그 구성을 사용하고, 없으면 기본 음색을 사용합니다.
  - Dialog에서 현재 채널별 음색 구성을 이름을 붙여 저장하고, 저장한 프리셋을 삭제할 수 있습니다.
  - `음색` 버튼 옆의 프리셋 콤보박스에서도 `자동 음색`과 저장한 프리셋을 바로 적용할 수 있습니다.
  - 전체 MML 탭에서는 `전체 음소거` 체크박스로 모든 채널을 한 번에 음소거/해제하고, 개별 채널 탭에서는 `채널 음소거`로 해당 채널만 재생에서 제외할 수 있습니다.
  - 설정은 브라우저 로컬 저장소에 보관되며, MML 재생과 나눠복사 악보 듣기에 적용됩니다.

  - 전체 MML은 파트 내용 글자 수 합계만 표시합니다. `MML@`, 쉼표, 세미콜론은 제외합니다.
  - 글자 수는 `1,234 자` 형식으로 표시하며, 앞에 파트 이름은 붙이지 않습니다.

  - Dialog에서 악보별 듣기/복사 버튼을 제공합니다.

- 파비콘 적용

## MIDI 변환 규칙

MIDI 변환은 먼저 파일 전체를 훑어 등장하는 악기를 분석합니다.
화면에서는 악기명 단위로 선택하고, 내부적으로는 원본 MIDI 채널/프로그램 정보를 유지합니다.

### MIDI 분석과 악기 선택

- 선택 UI는 악기명 목록을 보여줍니다. 같은 악기명은 하나의 선택지로 표시됩니다.
- 같은 악기명이 여러 채널에 있어도 사용자는 하나의 악기로 켜고 끕니다.
- 한 MML 채널에는 여러 악기를 선택할 수 있습니다.
- 여러 MML 채널이 같은 악기를 선택할 수 있습니다.
- 원본 MIDI 노트 하나는 전체 MML 채널 중 한 곳에만 배치됩니다. 같은 노트를 여러 MML 채널에 복제하지 않습니다.
- 단, 완전 중복 노트 병합은 원본 연주 경계를 지키기 위해 같은 채널/프로그램 상태 안에서만 수행합니다.
  - 기준: 64분음표 양자화 후 같은 음높이, 같은 시작 위치, 같은 종료 위치
  - 병합 시 velocity는 가장 큰 값을 기준으로 중복 수만큼 조금 보정합니다.
- MIDI 10번 채널의 타악 노트는 킥/스네어/심벌즈처럼 노트 번호별 비트 악기로 묶어 표시합니다.
- 드럼, 스네어, 북, 공, 심벌즈, 킥, 하이햇, 탐 등 비트 힌트가 있는 악기는 비트 그룹으로 분류합니다.
- 비트 악기 그룹이 없는 MIDI에서는 역할 선택에서 `비트` 항목을 표시하지 않고, 다이얼로그에 짧은 안내 문구를 표시합니다.

### MML 채널 설정

- Export 채널 수 기본값은 3개입니다.
- 기본 역할은 1번 채널 `고음`, 3번 채널 `저음`, 나머지 채널 `자동`입니다.
- 각 MML 채널은 개별적으로 다음 값을 가집니다.
  - 역할: `자동`, `고음`, `저음`, `비트`
  - 겹침 병합 방식: `모두`, `절반`, `안함`
  - 선택 악기 목록
- `자동`/`고음`/`저음` 채널은 일반 악기만 선택할 수 있습니다.
- `비트` 채널은 비트 그룹 악기만 선택할 수 있습니다.
- `비트` 역할은 오직 비트 그룹 노트만 수집합니다.
- `자동`/`고음`/`저음`은 비트 악기 후보를 가져가지 않습니다.

### 역할별 배치

- `자동`은 중립 역할이며, 배치 가능한 일반 악기 음을 보완적으로 가져갑니다.
- `고음`은 `O4` 이상, 즉 MIDI 60 이상 음을 우선 수집합니다.
  - `O4` 미만 음은 다른 적합한 채널에 배치할 수 있으면 먼저 양보합니다.
- `저음`은 `O4` 미만, 즉 MIDI 59 이하 음을 우선 수집합니다.
  - `O4` 이상 음은 다른 적합한 채널에 배치할 수 있으면 먼저 양보합니다.
- 같은 역할을 여러 Export 채널에 지정하면 채널 번호가 낮은 쪽이 먼저 더 높은 점수의 노트를 가져갑니다.
- 한 Export 채널은 단선율로 취급하므로 이미 음이 울리는 중이면 같은 파트에 새 음을 겹쳐 넣지 않습니다.
- 선택한 Export 채널 수보다 동시에 필요한 음이 많으면 남은 음은 생략될 수 있습니다.

### 겹침 병합

- 겹침 병합 방식은 기본적으로 멜로디와 화음2가 `절반`, 나머지 채널이 `모두`입니다.
- `모두`는 기존 체크박스를 켰을 때와 같은 방식으로, 겹치는 노트를 구제책으로 병합합니다.
- `안함`은 기존 체크박스를 껐을 때와 같은 방식으로, 겹침 병합을 사용하지 않습니다.
- `절반`은 앞 노트가 50% 이상 지난 뒤에 새 노트가 겹쳐 들어올 때만 병합합니다.
- 새 MIDI 노트가 들어올 때는 먼저 비어 있는 적합한 MML 채널에 정상 배치합니다.
- 정상 배치할 수 있는 채널이 없을 때만, 겹침 병합이 허용된 채널을 구제책으로 사용합니다.
- 겹침 병합이 적용되면 기존 노트를 새 노트 시작 위치에서 자르고, 새 노트를 원래 길이대로 배치합니다.
- 예: 기존 노트가 `0~4박`, 새 노트가 `1~5박`이면 기존 노트는 `0~1박`으로 줄고 새 노트가 `1~5박`으로 들어갑니다.
- 새 노트가 다른 선택 악기 채널에 정상 배치될 수 있다면 겹침 병합은 사용하지 않습니다.

## 실행

폴더 전체를 서버에 올려 사용하세요.
로컬 확인은 아래처럼 실행할 수 있습니다.

```bash
cd mabinogi_mml_public
python -m http.server 8000
```

브라우저에서 `http://localhost:8000/`로 접속합니다.
`index.html`만 따로 복사하면 기본 SF2와 JS 파일을 찾지 못하므로 폴더 구조를 유지해야 합니다.

- 전체 MML 표시 영역은 빈 채널 콤마를 유지해 파트 순서를 보존합니다.
- 전부복사/파일저장은 빈 채널을 제거한 정돈된 전체 MML을 사용합니다.
- 나눠복사는 파트 순서와 페이지별 독립 실행 상태를 보존하기 위해 필요한 빈 채널 위치를 유지할 수 있습니다.

## MML 자동 최적화 규칙

MID/MML 파일 불러오기, 전부복사, 파일저장, 쉼표 삭제, 시작 공백 추가, 나눠복사 결과에는 공통 최적화 단계를 적용합니다. MIDI 변환과 MML 붙여넣기/불러오기에서는 가장 먼저 울리는 노트 앞의 전체 공통 무음을 자동으로 제거합니다.

- 각 비어 있지 않은 채널은 시작 부분에 `V`, `O`, `L`을 명시합니다.
- 시작 템포가 없으면 멜로디 채널 시작에 `T120`을 추가합니다.
- 모든 `T` 템포 명령은 글로벌 규칙에 맞게 멜로디 채널로 이동합니다.
  - 다른 채널의 `T`는 같은 박자 위치에 맞춰 멜로디 채널로 옮깁니다.
  - 템포 위치가 멜로디 음표 중간이면 음표를 `&`로 나누어 같은 연주가 유지되도록 합니다.
- `N` 코드는 출력하지 않고 `a~g`, `+`, `-`, `O`, `<`, `>` 조합으로 다시 씁니다.
- 연속 쉼표는 같은 무음 길이를 유지하는 범위에서 더 짧은 `r` 표기로 합칩니다.
- 이미 `&`로 이어진 음은 가능한 짧은 길이 표기로 다시 씁니다.
- 같은 음이 연속되어 있어도 원래 `&`로 이어진 음이 아니면 임의로 합치지 않습니다.
- `T/O/L/V`는 대문자, `r/n/a~g`는 소문자로 정리합니다.

## 성능 최적화 메모

- 긴 MML을 재생할 때 모든 음표를 한 번에 예약하지 않고, 현재 재생 위치 기준 약 1.6초 앞까지만 예약하는 look-ahead 스케줄러를 사용합니다.
- SF2 샘플 버퍼와 음표별 재생 region은 재생 시작 전에 준비해 첫 재생 중 끊김을 줄입니다.
- V0처럼 소리 나지 않는 음표는 실제 오디오 노드를 만들지 않습니다.
- 64분음표처럼 짧은 음표는 attack/release 시간을 음 길이에 맞게 줄여 짧은 음이 뭉개지거나 밀리는 현상을 줄였습니다.
- 재생이 끝난 AudioBufferSourceNode는 자동으로 active 목록에서 제거해 장시간 재생 시 메모리 증가를 줄였습니다.


- 재생 시간 표시는 `분:초.밀리초` 형식으로 표시됩니다. 예: `01:23.456`


## 볼륨

볼륨 슬라이더는 기본 100%이며, 필요하면 최대 150%까지 증폭할 수 있습니다. 100%를 넘기면 악보와 사운드폰트에 따라 소리가 찢어질 수 있습니다.

- 재생 배속: 0.75x~1.50x 범위에서 조절할 수 있으며 기본값은 1.00x입니다.
